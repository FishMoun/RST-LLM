**Key Notes 1: Form of Constraints**

1. The allowed symbols in constraints include `<, <=, >, >=, ∧,(),abs()`.(`∨` `==`is not not allowed, action must be atomic)
   Forms like `a < b < c` are not allowed; only forms like `a < b ∧ b < c` are permitted.
2. Input constraints must not contain natural language, and expressions such as `ia(t)` are not allowed.
   Use `ia` directly instead.
3. Input constraints should contain **only** model input variables, control-point variables from the test configuration,
   Model output variables or other constructed variables are not allowed.
4. If a model input variable is parameterized into control-point variables, the input constraint should include **only** the control-point variables, without including the original variable.
5. Temporal constraints specify constraints on the actual duration of the action, for example: `duration <= k`, do not use `duration == k`
   where `duration` refers to the action duration, and `k` should be a constant or an input variable.
   If no specific constraint exists, use `true`.
6. Temporal constraints and test configuration constraints should **not appear** in the input constraint.
7. The test configuration may assign a constant value to an input variable; in this case, that variable does not need to be considered again in the input constraint.

**Key Notes 2: Principles of Action Definition**

1. The occurrence of an action **must not have preconditions**.
   For example, an action must not require the system to already be in a certain state.
   System states should be triggered by one or more actions or sequences of actions.
2. If symmetric signals exist among input variables, action merging is not allowed.
   A complete list of actions must be explicitly enumerated.
3. The input constraints of actions must not overlap.
   All actions in the action set must be pairwise mutually exclusive.

**Key Notes 3: Format Correctness**

1. If an action generated during analysis does not satisfy the required constraint form,
   that action must **not** be included in the final action set.
   In this task, **output format correctness has the highest priority**.
