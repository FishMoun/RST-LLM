Model Description:
The Triplex Signal Monitor is a redundancy management system model designed for airborne applications. It monitors three redundant input signals (ia, ib, ic) from three branches (A, B, C) to detect and isolate failures using a trip level (Tlevel) and persistence count limit (PClimit). The model operates in no-fail and single-fail states, selecting an output value (set_val) via mid-value selection in the no-fail state and good channel average in the single-fail state. It also tracks persistence count (PC), totalizer count (TC), and fault code (FC) to manage failure reporting and signal selection.

Requirement Item 1:
    Requirement Description:
    In the no-fail state, a miscompare that lasts beyond the persistence limit shall be reported as a failure.
    Original Text:
    [RM-001] In the no-fail state, a mis-compare, which shall be characterized by one branch differing with the other two branches by a unique trip level that lasts for more than the persistence limit, shall be reported to failure management as a failure.

Requirement Item 2:
    Requirement Description:
    In the no-fail state, the mid-value shall be the selected value, and a first failure in progress does not affect the selection method.
    Original Text:
    [RM-002] In the no-fail state, the mid-value shall be the selected value.  Note: a first failure in progress will not affect the method for determining the selected value.

Requirement Item 3:
    Requirement Description:
    In the single fail state, the selected value shall be the good channel average of the two remaining good branches.
    Original Text:
    [RM-003] In the single fail state, a good channel average of the remaining two good branches shall be used to determine the selected value.

Requirement Item 4:
    Requirement Description:
    If a second failure is in progress, the selected value shall remain unchanged from the previous selected value.
    Original Text:
    [RM-004] If a second failure is in progress, the selected value shall remain unchanged from the previous selected value.
